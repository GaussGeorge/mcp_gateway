### 一、 轻量级工具矩阵（老鼠流 / 资源消耗极低）

这类工具的特点是：执行速度极快（几毫秒到几十毫秒），几乎不消耗本地 CPU 和 GPU。它们在并发时用来测试网关的**基础路由吞吐量（Throughput）**和**框架自身的开销**。

1. **精确数学计算器 (Calculator MCP)**
* **真实场景**：Agent 在规划财务报表或进行逻辑推理时，调用它进行不会产生“幻觉”的精确加减乘除。
* **系统特征**：纯 CPU 极轻量级计算，耗时 < 1ms。


2. **实时天气与时间查询 (Weather & Time MCP)**
* **真实场景**：大模型获取当前的物理世界状态。
* **系统特征**：极轻量的网络 I/O 密集型。主要受外部 API 响应时间影响。


3. **网页信息检索 (Web Search MCP)**
* **真实场景**：Agent 搜索最新新闻或开源文档以补充知识库。
* **系统特征**：网络 I/O 密集型，会有不可控的外部网络延迟波动（几十到几百毫秒），可以测试你的网关对外部长连接的管理能力。


4. **基础文本正则与格式化 (Text Formatting MCP)**
* **真实场景**：将杂乱的 JSON 或日志数据进行初步清洗。
* **系统特征**：轻度 CPU 消耗。



### 二、 重量级工具矩阵（大象流 / 资源争抢严重）

这类工具的特点是：耗时长（数秒甚至数十秒），会瞬间榨干某一种硬件资源（GPU 算力、显存带宽或 CPU 核心）。它们用来测试网关的**动态定价阻断能力**和**防雪崩能力**。

1. **核心重载：大模型深度推理与总结 (LLM Reasoner MCP)**
* **真实场景**：Agent 抓取了一篇 5000 字的长文，要求提取核心摘要并翻译。
* **系统特征**：**极端 GPU 算力与显存带宽消耗**。耗时与请求的 Token 数量严格正相关（如你之前测试的 ~10秒）。这是触发 Rajomon 网关启动 Overload 保护的绝对主力。


2. **内存杀手：长文档向量化 (Document Embedding MCP)**
* **真实场景**：企业级 RAG 知识库构建。Agent 需要把一本几十 MB 的 PDF 手册灌入向量数据库。
* **系统特征**：**极端内存/显存爆发消耗 (Memory Burst)**。它不仅耗时，还可能在瞬间申请大量显存导致系统 OOM，用来测试网关瞬间调高 Price 以拒绝后续重载请求的反应速度。


3. **阻塞队列杀手：Python 沙盒数据分析 (Python Sandbox MCP)**
* **真实场景**：用户丢给 Agent 一个包含 50 万条销售记录的 CSV 文件，要求分析趋势并画出折线图。
* **系统特征**：**长时间的 CPU 独占与队列阻塞**。这类请求会把后端的 Worker 线程长时间卡死。用来验证网关能否在后端排队严重时，保障轻量级计算器请求依然能畅通无阻（这就是 Fairness 公平性）。



### 三、 终极武器：基准对照工具

1. **纯测试桩 (Mock Heavy Tool)**
* **功能**：接收 `cpu_burn_ms` 和 `memory_mb` 参数，精准控制消耗。
* **作用**：真实的大模型耗时有波动。当你需要画出极度平滑的 P95/P99 延迟 CDF 曲线（如你之前上传的 `latency_cdf.png`）时，用这个纯数学模拟的重载工具，能排除一切硬件抖动干扰，呈现最完美的算法对比。
